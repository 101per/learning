# transformer的应用都有啥？比如用在了哪些模型上面？我了解基础的transformer模型，我还想深入了解它的发展和应用。

既然你已经掌握了 Transformer 的基础（Encoder-Decoder 架构、Self-Attention 等），那我们直接跳过入门科普，聊聊它是如何从一个“翻译工具”演变成现代 AI 的“通用骨架”的。

Transformer 的发展可以看作是一场“注意力机制”对各种领域的全面入侵。

------

## 1. 自然语言处理 (NLP)：从理解到生成

这是 Transformer 的老家。根据架构侧重点不同，分化出了三大流派：

- **Encoder-only (理解派):** * **代表模型：** BERT, RoBERTa, ALBERT。
  - **应用：** 擅长文本分类、情感分析、命名实体识别。它像是一个深思熟虑的读者，通过双向注意力看清上下文。
- **Decoder-only (生成派 - 目前的主流):** * **代表模型：** GPT 系列 (GPT-3, GPT-4), Llama, Claude, Mistral。
  - **应用：** 写作、编程、对话。现在的“大模型”基本都是这个路线，因为它在处理超大规模参数和自回归生成时效率极高。
- **Encoder-Decoder (全能派):** * **代表模型：** T5 (Text-to-Text Transfer Transformer), BART。
  - **应用：** 文本摘要、翻译。它将所有任务都统一成“输入一段文字，输出一段文字”的模式。

------

## 2. 计算机视觉 (CV)：Vision Transformer (ViT)

以前视觉领域是 CNN（卷积神经网络）的天下，但 2020 年 **ViT** 的出现打破了僵局。

- **核心逻辑：** 它把图片切成一个个小方块（Patches），就像对待单词一样，把这些方块喂给 Transformer。
- **代表模型：** ViT, Swin Transformer (引入了滑动窗口，解决了高分辨率计算量大的问题)。
- **应用：** 图像识别、目标检测、医学影像分析。

------

## 3. 多模态与生成艺术：打破次元壁

这是目前最火的领域，让 AI 能同时听、看、说。

- **CLIP (Contrastive Language-Image Pre-training):** 连接文本和图像的桥梁。它是 DALL-E 和 Stable Diffusion 的灵魂。
- **DALL-E / Midjourney:** 利用 Transformer 处理文本语义，结合扩散模型生成图像。
- **Sora:** 视频生成。Sora 的核心是 **Diffusion Transformer (DiT)**，它证明了 Transformer 架构在处理视频序列这种“长数据”时依然强悍。

------

## 4. 科学发现与工业应用

Transformer 已经开始解决人类几十年没搞定的硬核科学问题：

- **AlphaFold 2:** 预测蛋白质三维结构。它将氨基酸序列视为“语言”，利用 Transformer 建模空间关系，彻底改变了结构生物学。
- **代码自动补全：** 如 GitHub Copilot (基于 Codex 模型)。
- **自动驾驶：** 特斯拉的 FSD 方案中，利用 Transformer 将多个摄像头的 2D 图像转换成 3D 向量空间（Occupancy Network）。

------

# 请分别给我介绍一下，Clip,Deit,Mae,Swin,Vit

既然你已经对 Transformer 的基础有所了解，这五个模型正是你深入“视觉 Transformer (Vision Transformer)”及其衍生物的最佳切入点。

它们分别代表了 **纯视觉架构 (ViT)**、**训练优化 (DeiT)**、**层次化结构 (Swin)**、**自监督学习 (MAE)** 以及 **跨模态连接 (CLIP)**。

------

## 1. ViT (Vision Transformer) —— 开山之作

**核心贡献：** 证明了不需要卷积（CNN），只靠纯 Transformer 也能搞定图像识别。

- **原理：** 它把一张图片切成 $16 \times 16$ 的小方块（Patches），每个方块拉平成一个向量，看作一个“单词”，然后加上位置编码，直接丢进标准的 Transformer Encoder。
- **痛点：** 它是“数据饕餮”，如果没有海量数据（如 JFT-300M）预训练，效果不如传统的 ResNet。
- **地位：** 视觉领域的“初代机”，确立了图片序列化的范式。

## 2. DeiT (Data-efficient Image Transformers) —— 训练大师

**核心贡献：** 解决了 ViT 必须依赖海量数据的问题，让 Transformer 在小数据集（如 ImageNet-1K）上也能起飞。

- **核心技术：** **Distillation Token (蒸馏算子)**。
- **逻辑：** 引入了一个老师（通常是强大的 CNN 模型）。DeiT 在训练时，不仅学习图片标签，还专门学习老师的“看法”。通过这种“名师指导”，DeiT 用更少的数据就达到了极高的精度。
- **意义：** 让普通实验室和公司也能练得起视觉 Transformer。

## 3. Swin Transformer —— 阶层式霸主

**核心贡献：** 解决了 ViT 计算量大、难以处理高分辨率图片的问题，成为了目前的 CV 骨干网络（Backbone）首选。

- **核心技术：** **Shifted Windows (移动窗口)**。
- **逻辑：**
  - 不像 ViT 那样全局计算注意力（计算量随分辨率平方增长），Swin 只在局部小窗口内算注意力。
  - 通过“移动窗口”让相邻的窗口产生信息交互。
  - 引入了类似 CNN 的**层级结构**（由粗到细），这让它非常适合做目标检测和语义分割。
- **地位：** 目前计算机视觉领域最实用、最全能的 Transformer 模型之一。

## 4. MAE (Masked Autoencoders) —— 自监督黑马

**核心贡献：** 证明了“完形填空”在视觉领域同样强大，由何恺明大神提出。

- **原理：**
  1. 把图片随机**遮住 75%** 的区域（只剩下一点点碎块）。
  2. 让 Encoder 只处理没被遮住的部分（省算力）。
  3. 让 Decoder 尝试把整张图画回来。
- **震撼点：** 这种“遮盖学习”迫使模型理解图片的语义结构，而不仅仅是像素。
- **应用：** 它提供了一种极其强大的**预训练方法**，先用 MAE 在海量无标注图片上练手，再去微调其他任务，效果惊人。

## 5. CLIP (Contrastive Language-Image Pre-training) —— 破壁人

**核心贡献：** 打通了文字和图片的边界，是 DALL-E 和 Stable Diffusion 的底层基石。

- **原理：** **对比学习**。它同时拥有一个文本编码器和一个图像编码器。
- **训练目标：** 给它几亿对“图片+文字描述”，让它学习：如果图片里有条狗，对应的文字向量和图片向量在空间上必须靠得近；无关的则要离得远。
- **超能力：** **Zero-shot (零样本) 能力**。你不需要教它什么是“宇航员骑马”，只要给它一张图和这句话，它就能判断它们是否匹配。
- **地位：** 开启了多模态 AI 时代，让机器真正具备了“看图识字”的常识。

------

### 五大模型对比表

| **模型** | **核心标签** | **解决的问题**              | **关键特征**            |
| -------- | ------------ | --------------------------- | ----------------------- |
| **ViT**  | 纯血派       | 验证 Transformer 可用于视觉 | Patch 嵌入 + 全局注意力 |
| **DeiT** | 优化派       | 减少对海量数据的依赖        | 知识蒸馏 (Distillation) |
| **Swin** | 实用派       | 高分辨率图像计算量大        | 层级结构 + 移动窗口     |
| **MAE**  | 自监督       | 无标注数据的特征提取        | 遮掩重建 (类似 BERT)    |
| **CLIP** | 跨模态       | 图像与文本的语义对齐        | 对比学习 + 零样本识别   |

# 那有没有一个模型把DeiT，Swin的思想一起使用呢？

啊，这个问题很有意思！你问的是有没有把 **DeiT（Data-efficient Image Transformer）** 和 **Swin Transformer（Shifted Window Transformer）** 的思想结合在一起的模型。我们可以先拆开来看，再讨论是否有已有尝试，或者可能的结合方式。

------

### 1️⃣ 回顾两个模型的核心思想

**DeiT**

- 基于 **ViT（Vision Transformer）** 的改进，专注于 **数据高效训练**。
- 核心特点：使用 **知识蒸馏**（distillation token）和一些训练策略，使 Transformer 在中小规模数据集上也能表现好。
- 结构上还是 **全局 self-attention**。

**Swin Transformer**

- 核心是 **局部窗口 self-attention + 移动窗口 (Shifted Window)**。
- 优点：降低计算复杂度（从 O(N²) 降到 O(M²)），引入 **层级特征表示**，更像 CNN 的多尺度特征。
- 对大图像或高分辨率任务非常有效。

------

### 2️⃣ 结合的思路

你希望的“结合”，可以理解为：

- 保留 **Swin 的局部+层次结构优势**
- 同时用 **DeiT 的数据高效训练和蒸馏机制**

理论上有两条路：

1. **在 Swin 基础上加入 DeiT 蒸馏机制**
   - Swin 本身是大模型，训练时可以加入 **distillation token** 或者用一个 **教师模型指导训练**。
   - 有研究把 ViT 的蒸馏方法直接加到 Swin 上，这样可以在中小数据集上训练 Swin 变体。
2. **Hybrid 模型**
   - 前几层用 **Swin 的局部 window attention** 来提取局部特征
   - 后几层用 **DeiT/ViT 的全局 attention** 聚合全局信息
   - 结合 **知识蒸馏**，提升数据效率。
   - 有些论文称之为 **“Local-Global Vision Transformer”** 或 **“Hierarchical + Distillation ViT”**。

------

### 3️⃣ 相关论文/模型示例

我查到一些比较接近你想法的工作：

| 模型/论文                                        | 思路                                       | 特点                         |
| ------------------------------------------------ | ------------------------------------------ | ---------------------------- |
| **T2T-ViT (Tokens-to-Token ViT)**                | 类似 DeiT 的数据高效 + 局部 token 聚合     | 部分局部感受野，适合小数据集 |
| **CaiT (Class-Attention in Image Transformers)** | DeiT 的蒸馏 + 大型 ViT 变体                | 专注于 class token 的蒸馏    |
| **Swin + Distillation**（非官方实现）            | Swin 训练中加入 DeiT 的 distillation token | 结合了层次结构和数据高效训练 |
| **PVT (Pyramid Vision Transformer)**             | 层次化设计 + 可选蒸馏训练                  | 类似 Swin，但更灵活          |

------

✅ **总结**
目前没有一个标准叫做 “DeiT-Swin”，但从方法上完全可以把它们结合：

- Swin 的局部窗口 & 层次结构
- DeiT 的知识蒸馏 & 数据高效训练
  这个思路在社区里有实现，但大多数是自定义实验。

# 那这么说Swin设置的模式对训练没有什么提升？只对处理有提升吗？

差不多可以这样理解，但细节要分得很清楚。我们慢慢拆开来看：

------

### 1️⃣ Swin Transformer 的“模式”主要优势

Swin 的设计核心是 **局部窗口 self-attention + 移动窗口 + 层次结构**：

| 特性                       | 对训练的影响                                | 对推理/处理的影响                             |
| -------------------------- | ------------------------------------------- | --------------------------------------------- |
| 局部窗口注意力             | 可以加快梯度计算（相比全局 self-attention） | 显著减少计算量，能处理大图                    |
| 移动窗口（Shifted Window） | 提供跨窗口的信息交流，训练稳定性略提升      | 增强感受野，减少全局 attention 的计算压力     |
| 层次结构 (Hierarchical)    | 有助于梯度传递，训练更平滑，但不是根本提升  | 自然输出多尺度特征，方便下游任务（检测/分割） |

总结：**Swin 的设计主要是为了提高计算效率和可扩展性**，尤其是对高分辨率图像；对训练收敛速度或数据效率的提升相对有限。

------

### 2️⃣ 为什么训练提升不明显

- **DeiT 的改进**（数据高效 + 蒸馏）是专门针对训练收敛和小数据集的。
- **Swin 的局部注意力**只是降低了每层的计算量，但梯度质量、收敛速度、对数据量的需求并没有本质改变。
- 换句话说，Swin 更像是 **训练成本降低 + 推理成本降低**，而不是**训练效果提升**。

------

### 3️⃣ 实际应用场景

- 如果你 **数据量少或需要快速收敛** → DeiT 的蒸馏和优化技巧更关键。
- 如果你 **图像分辨率高、计算资源有限** → Swin 的窗口机制能让你训练/推理更高效。
- 如果你把两者结合 → 既能处理大图，又能在小数据集上高效训练。

------

一句话总结：

> **Swin 改进的是“怎么算”而不是“训练更好”；DeiT 改进的是“怎么训练”而不是“算更快”。**

# 一、Swin Transformer真正解决的问题

在 Vision Transformer 里，self-attention 是 **全局 attention**：

[
O(N^2)
]

如果图像是：

```
224 × 224
patch = 16
token = 196
```

还好。

但如果是：

```
1024 × 1024
patch = 4
token ≈ 65536
```

attention 就变成：

[
65536^2
]

这是 **无法训练的级别**。

所以 Swin Transformer 的核心目标其实是：

**让 Transformer 可以处理高分辨率视觉任务。**

例如：

- 目标检测
- 语义分割
- 视频
- 医学图像

这些任务都需要 **高分辨率特征图**。

------

# 二、Swin 的三个核心设计

## 1 Window Attention

只在局部 window 计算 attention

例如：

```
8 × 8 window
```

复杂度变成：

[
O(N × M^2)
]

M = window size

这样复杂度接近 CNN。

------

## 2 Shifted Window

如果只做 window attention 会有问题：

```
|A|B|
|C|D|
```

A 看不到 B/C/D。

于是下一层：

```
 shift window
```

变成

```
|A B|
|C D|
```

于是 **跨窗口信息传播**。

这一步是 Swin 的关键创新。

------

## 3 Hierarchical Structure

像 CNN 一样：

```
stage1  patch=4
stage2  patch=8
stage3  patch=16
stage4  patch=32
```

逐层 **降低分辨率 / 增加 channel**。

这对：

- detection
- segmentation

非常重要。

因为这些任务需要 **多尺度特征**。

------

# 三、Swin 对训练有没有帮助？

你之前说：

> “只对处理有提升”

其实更准确的说法是：

| 方面       | 影响       |
| ---------- | ---------- |
| 训练稳定性 | 有一定帮助 |
| 训练速度   | 略有提升   |
| 数据效率   | 几乎没有   |
| 模型能力   | 提升明显   |

------

### 1 训练稳定性

局部 attention 有一个副作用：

**优化更容易。**

原因：

全局 attention 的梯度非常 noisy。

局部 attention 类似 CNN 的 **local inductive bias**。

所以训练会更稳定。

------

### 2 模型能力

Swin 在很多任务上比 ViT 更强：

例如：

- COCO detection
- ADE20K segmentation

原因：

**层次结构 + 多尺度特征**

这对视觉任务非常关键。

------

# 四、为什么 DeiT 解决的是另一类问题

Training data-efficient image transformers & distillation through attention (DeiT)

解决的问题是：

**ViT 需要超大数据集。**

原始 ViT 用的是：

```
JFT-300M
```

而 DeiT 只用：

```
ImageNet-1K
```

方法：

1️⃣ distillation token
2️⃣ 强数据增强
3️⃣ teacher model (CNN)

所以 DeiT 是：

```
training trick
```

而 Swin 是：

```
architecture design
```

# 请介绍inductive bias（归纳偏置）

**Inductive Bias（归纳偏置）**是机器学习里一个非常核心、但常常被忽视的概念。理解它，你会更容易理解为什么 **CNN、Transformer、Swin、MAE** 这些模型要那样设计。

我从 **直觉 → 数学 → 在深度学习中的体现 → 在 ViT / CNN 中的差异** 四个层次给你讲。

------

# 一、直觉理解：模型“先天的世界观”

**Inductive Bias = 模型在学习之前，对世界做出的假设。**

因为现实情况是：

> 数据永远不够。

如果一个模型**没有任何假设**，它会面临一个问题：

同一组训练数据，可以对应 **无限多种函数**。

模型根本不知道选哪个。

于是我们必须给模型一些 **偏好**。

这就是 **Inductive Bias**。

------

### 举个简单例子

假设我们有数据：

```
x: 1 2 3
y: 2 4 6
```

我们要预测：

```
x = 4
```

理论上可能的函数有：

```
y = 2x
y = 2x + sin(100x)
y = 2x + 1000*(x-1)(x-2)(x-3)
```

这些函数 **都能 perfectly fit 数据**。

但人类会选：

```
y = 2x
```

为什么？

因为我们有一个 **归纳偏置**：

> “自然规律通常是简单的。”

这就是 **Occam’s Razor（奥卡姆剃刀）**。

------

# 二、数学角度

在机器学习中，我们学习的是一个函数：

[
f_\theta(x)
]

如果没有 inductive bias：

学习问题是：

[
\text{choose } f \in \mathcal{F}
]

函数空间无限大。

而 inductive bias 做的是：

[
\text{choose } f \in \mathcal{F'} \subset \mathcal{F}
]

也就是说：

**限制模型可能学习到的函数空间。**

------

# 三、深度学习中的 Inductive Bias

在深度学习里，inductive bias 主要来自三个地方：

| 来源       | 例子                 |
| ---------- | -------------------- |
| 模型结构   | CNN、Transformer     |
| 参数共享   | convolution          |
| 局部性假设 | receptive field      |
| 训练目标   | contrastive / masked |

------

# 四、CNN 的 Inductive Bias

最经典的例子是 **卷积神经网络 CNN**。

例如：

ImageNet Classification with Deep Convolutional Neural Networks

CNN 的归纳偏置有三个：

### 1 局部性（Locality）

假设：

> 图像的有用信息在局部区域。

例如：

```
边缘
角点
纹理
```

所以 CNN 使用：

```
3×3 kernel
```

只看局部。

------

### 2 平移不变性（Translation Equivariance）

假设：

> 一只猫在左边和右边是同一只猫。

所以 CNN 使用：

```
weight sharing
```

同一个卷积核扫描整张图。

------

### 3 层级结构（Hierarchy）

假设：

```
edge → texture → part → object
```

这就是 CNN 的层级结构。

------

# 五、Transformer 的 Inductive Bias（非常弱）

原始 Transformer：

Attention Is All You Need

几乎没有视觉假设。

每个 token：

```
可以和任何 token 交互
```

这意味着：

模型必须 **从数据中学会一切**：

- 什么是局部
- 什么是物体
- 什么是结构

所以它需要：

```
海量数据
```

这就是为什么

**ViT 在小数据集上输给 CNN。**

------

# 六、ViT vs CNN 的关键区别

| 模型 | Inductive Bias |
| ---- | -------------- |
| CNN  | 强             |
| ViT  | 弱             |

CNN 假设：

```
图像是局部的
```

ViT 假设：

```
图像是token集合
```

结果就是：

| 数据规模 | CNN  | ViT  |
| -------- | ---- | ---- |
| 小数据   | 更好 | 较差 |
| 大数据   | 一般 | 更强 |

这在：

An Image is Worth 16x16 Words

论文里被证明。

# 七、Swin Transformer 的本质

现在你就能理解：

为什么要有 **Swin Transformer**。

Swin Transformer

Swin 做了一件事：

**给 Transformer 加回一点 inductive bias。**

具体是：

| 设计             | 引入的 bias |
| ---------------- | ----------- |
| window attention | 局部性      |
| shift window     | 邻域关系    |
| hierarchical     | 多尺度结构  |

所以：

```
CNN bias 强
Swin bias 中等
ViT bias 弱
```

# 八、MAE 的另一个思路

Masked Autoencoders Are Scalable Vision Learners

MAE 不是加结构 bias。

而是加 **training bias**：

```
mask 75% patch
```

假设：

> 图像具有冗余结构。

所以模型必须学会：

```
语义理解
```

# 九、CLIP 的归纳偏置

CLIP: Learning Transferable Visual Models From Natural Language Supervision

CLIP 的 inductive bias 是：

```
图像和文本语义一致
```

用 **contrastive learning** 强制对齐。


# 用 contrastive learning 强制对齐。我经常看到对齐这个词，但是我并不清楚他们是不是意思一样，我不懂对齐在数学，或者深度学习里的意思是什么

这是一个**非常好的问题**。
“**对齐（alignment）**”这个词在深度学习里确实被频繁使用，但很多文章不会严格定义它，导致看起来有点“玄学”。实际上它在数学上是**很具体的概念**。

我给你从 **直觉 → 数学表达 → 在 contrastive learning 中 → 在不同领域中的“对齐”含义** 一步步讲清楚。

------

# 一、最直观的理解：让两个表示“靠近”

在深度学习中，“对齐”通常指：

> **让两个本应表达同一语义的向量，在表示空间中变得接近。**

假设我们有两个编码器：

- 图像编码器 ( f(x) )
- 文本编码器 ( g(t) )

例如在
CLIP: Learning Transferable Visual Models From Natural Language Supervision

我们会得到两个向量：

[
v = f(\text{image})
]

[
u = g(\text{text})
]

如果图片内容是：

```text
a dog running
```

那么训练目标就是：

[
v \approx u
]

也就是说：

**让图像向量和文本向量“对齐”。**

------

# 二、数学上怎么实现“对齐”

通常用 **相似度函数（similarity）**。

最常见的是：

### 1 cosine similarity

[
sim(u,v) =
\frac{u \cdot v}{||u|| , ||v||}
]

如果两个向量完全一样：

```text
sim = 1
```

如果完全相反：

```text
sim = -1
```

训练目标就是：

```text
正确匹配 → similarity 高
错误匹配 → similarity 低
```

------

### 2 对比损失（contrastive loss）

CLIP 使用的是：

[
L = - \log
\frac{e^{sim(u_i,v_i)/\tau}}
{\sum_j e^{sim(u_i,v_j)/\tau}}
]

直觉理解：

```text
正确图片-文本 pair
→ 拉近

错误 pair
→ 推远
```

所以 embedding 空间会变成：

```text
dog image   → close to "dog"
cat image   → close to "cat"
car image   → close to "car"
```

这就是 **语义对齐**。

------

# 三、从几何角度理解“对齐”

假设 embedding 空间是二维：

```text
           dog text
              ↑
              |
dog image ----•---- cat image
              |
              ↓
           cat text
```

训练前：

```text
image space 和 text space 完全不同
```

训练后：

```text
dog image ≈ dog text
cat image ≈ cat text
```

两个空间的语义结构 **重合**。

这就叫：

> **embedding space alignment**

------

# 四、为什么要做对齐？

因为很多任务需要：

**跨模态理解**

例如：

| 任务                 | 需要什么                  |
| -------------------- | ------------------------- |
| image search by text | text→image alignment      |
| image caption        | image→text alignment      |
| multimodal LLM       | vision→language alignment |

例如：

```text
query: "a dog running"
```

系统会找：

```text
embedding closest image
```

而不是分类。

------

# 五、对齐其实有两种不同含义（非常重要）

在深度学习论文里，“alignment”其实有 **两种完全不同的用法**。

## 1 表示对齐（Representation Alignment）

这是我们刚讲的。

例如：

- CLIP
- multimodal model
- speech-text model

本质：

```text
不同模态 → 同一语义空间
```

------

## 2 模型行为对齐（AI Alignment）

在大模型领域：

“alignment”是另一个意思：

让模型行为符合人类意图。

例如：

- RLHF
- 安全训练

例如：

Training language models to follow instructions with human feedback

这里的 alignment 是：

```text
model output
≈ human preference
```

这和 embedding alignment **完全不是一个概念**。

------

# 六、Contrastive Learning 中的 Alignment

在对比学习领域，有一个经典理论：

**alignment vs uniformity**

来自论文：

Understanding Contrastive Representation Learning through Alignment and Uniformity on the Hypersphere

他们提出：

好的 representation 应该满足：

### 1 Alignment

相似样本距离小

[
||z_i - z_j||^2
]

例如：

```text
same object
same caption
same image augmentation
```

------

### 2 Uniformity

所有 embedding 均匀分布在空间中。

否则会：

```text
embedding collapse
```

例如：

所有向量变成：

```text
(0,0,0,0,0)
```

那就没信息了。

------

# 七、CLIP 的对齐其实做了两件事

CLIP loss 同时实现：

```text
Alignment
+
Uniformity
```

因为：

```text
positive pair → 拉近
negative pair → 推远
```

所以 embedding 会变成：

```text
语义聚类
```

例如：

```text
dog cluster
cat cluster
car cluster
```

------

# 八、一个非常简洁的总结

在深度学习里：

**对齐 = 让语义相同的表示在向量空间中靠近。**

数学上：

```text
maximize similarity(x,y)
```

几何上：

```text
distance(x,y) ↓
```

在 contrastive learning 中：

```text
positive pairs → closer
negative pairs → farther
```

# 一、强 inductive bias 的优点和问题

先看一个极端例子：**CNN**。

经典论文：
ImageNet Classification with Deep Convolutional Neural Networks

CNN 的 inductive bias 很强：

| 设计           | 假设                    |
| -------------- | ----------------------- |
| 卷积           | 图像是局部结构          |
| weight sharing | 物体位置不重要          |
| 层级结构       | edge → texture → object |

这些假设非常符合 **自然图像**。

所以 CNN 在 **小数据集** 上表现很好。

例如：

```text
ImageNet (1M)
```

CNN 可以很好训练。

------

## 但问题来了

如果任务不是自然图像呢？

例如：

- 文本
- 视频
- 音频
- 3D point cloud
- DNA
- 蛋白质

CNN 的假设就变成：

```text
错误的先验
```

模型会被 **限制住表达能力**。

------

# 二、强 inductive bias = 强限制

数学上可以这样理解：

模型学习函数：

[
f_\theta(x)
]

但 inductive bias 实际上限制了函数空间：

[
f_\theta(x) \in \mathcal{F}_{bias}
]

例如 CNN：

```text
只能表达“局部结构”
```

而真实世界可能需要：

```text
全局关系
长距离依赖
复杂交互
```

于是模型上限被限制。

------

# 三、Transformer 的革命

Transformer 出自：

Attention Is All You Need

它做了一件非常大胆的事情：

**几乎移除所有结构假设。**

Self-Attention：

[
Attention(Q,K,V) =
softmax(\frac{QK^T}{\sqrt{d}})V
]

含义是：

```text
每个 token
都可以和任何 token 交互
```

没有：

```text
局部限制
空间结构
层级假设
```

所以 Transformer 的 inductive bias **非常弱**。

------

# 四、弱 inductive bias 的代价

弱 bias 会带来一个问题：

**数据需求巨大。**

因为模型必须 **自己学习结构**。

例如：

ViT 论文：

An Image is Worth 16x16 Words

发现：

```text
ImageNet (1M)
→ CNN 更好

JFT-300M
→ ViT 更好
```

原因就是：

```text
data 足够大
→ bias 不再必要
```

------

# 五、为什么大模型时代反而减少 bias

核心原因其实是：

> **数据规模增长速度 > 人类设计 bias 的能力**

现代 AI 的训练数据：

```text
互联网
书籍
代码
视频
图像
```

规模：

```text
10^12 tokens
```

在这种规模下：

**模型可以自己学到结构。**

例如：

Transformer 会自动学到：

- 语法
- 语义
- 逻辑
- 世界知识

这些结构 **不是人为写进去的**。

------

# 六、Scaling Law 的影响

关键理论来自：

Scaling Laws for Neural Language Models

发现：

模型性能满足：

[
Loss \propto N^{-\alpha}
]

N 是：

```text
数据
参数
计算
```

关键发现：

**更大的模型 + 更多数据 = 更好性能**

如果模型 bias 很强：

```text
capacity 被限制
```

就无法继续 scaling。

所以：

**弱 bias → 更好的 scaling**
