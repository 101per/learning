# vit是vision transformer,是计算机CV方向用到transformer的技术吗？

你说得非常准确！**ViT (Vision Transformer)** 确实是将原本用于 NLP（自然语言处理）领域的 Transformer 架构成功引入到 CV（计算机视觉）方向的里程碑式技术。

简单来说，在 ViT 出现之前，计算机视觉基本上是 **CNN（卷积神经网络）** 的天下；而 ViT 的出现证明了：**即便不使用卷积，只靠注意力机制（Attention），也能在图像识别上取得顶尖效果。**

------

### ViT 的核心逻辑：把图片当成“文章”来读

Transformer 最初是为处理文字序列设计的（比如把一句话里的每个词作为一个单元）。ViT 的天才之处在于它巧妙地把图像转换成了序列：

1. **图像分块 (Patching)：** 它把一张图片像切披萨一样，切成一个个固定大小的小方块（Patches）。例如，一张 $224 \times 224$ 的图片可以切成 $14 \times 14$ 个小块。
2. **线性投影 (Linear Projection)：** 把每个小方块打平，转换成一个向量（特征向量）。
3. **位置编码 (Position Embedding)：** 关键步骤！因为 Transformer 本身不像 CNN 那样自带空间位置感，所以必须给每个小方块加一个“编号”，告诉模型哪块在左上角，哪块在右下角。
4. **Transformer 编码：** 像处理单词序列一样，把这些带编号的图像块丢进 Transformer 层，利用 **自注意力机制 (Self-Attention)** 让每个图像块都去观察其他所有图像块，从而捕捉全局联系。

------

### 为什么 ViT 这么火？（对比 CNN）

| **特性**     | **CNN (传统卷积)**                     | **ViT (Vision Transformer)**                                 |
| ------------ | -------------------------------------- | ------------------------------------------------------------ |
| **视野范围** | **局部性**：卷积核每次只看一小块区域。 | **全局性**：第一层就能看到整张图的所有像素。                 |
| **归纳偏置** | 强：自带“像素邻近”假设，小样本也能练。 | 弱：不预设像素规律，全靠数据自学。                           |
| **数据需求** | 中等。                                 | **极大**：通常需要海量数据（如 ImageNet-21k 或 JFT-300M）预训练才能超越 CNN。 |
| **扩展性**   | 随着模型变大，性能提升趋于平缓。       | **Scaling Law**：模型越大、数据越多，潜力几乎没有上限。      |

------

### 现在的现状

现在的 CV 领域已经不是“二选一”了，而是**全面开花**：

- **纯 Transformer 流派：** 像 ViT、DeiT 等，在超大规模数据集上霸榜。
- **层次化 Transformer：** 比如 **Swin Transformer**，它引入了类似 CNN 的滑动窗口思想，目前是很多视觉任务（如检测、分割）的主流骨干网络。
- **多模态：** 像现在的 GPT-4V 或 DALL-E，底层基本都是 Transformer，因为这样可以非常方便地让图片和文字共用一套“语言”。

**总结一下：** ViT 彻底打破了视觉和语言之间的壁垒，让“万物皆可 Transformer”成为了可能。

# 线性投影 (Linear Projection)： 把每个小方块打平，转换成一个向量（特征向量）。这一步是什么流程呢？

这一步其实是把“视觉信号”转化为“数字语言”的关键桥梁。在 Transformer 的世界里，它只认一维的向量（Vector），不认三维的图像张量（Height $\times$ Width $\times$ Channels）。

为了让 Transformer 读懂图片，**线性投影 (Linear Projection)** 实际上执行了两个动作：**展平 (Flattening)** 和 **特征映射 (Mapping)**。

------

### 具体的执行流程

假设你有一张彩色图片，切出的其中一个图像块（Patch）的大小是 $16 \times 16$ 像素，通道数为 $3$（RGB）。

#### 1. 展平 (Flattening)

首先，我们将这个 $16 \times 16 \times 3$ 的三维小方块直接“拉直”成一排数字。

- 计算方法：$16 \times 16 \times 3 = 768$。
- 结果：原本的小方块变成了长为 **768** 的一维向量。

#### 2. 线性映射 (Linear Mapping)

如果你直接用这 768 个原始像素值去算注意力，效果会很差，因为像素值对光照、噪声太敏感。

于是，我们给这组数字配一个**可学习的权重矩阵 $W$**。

- 通过矩阵乘法（即全连接层），将这 768 维的向量投影到一个固定的维度 $D$（例如 $D=512$ 或 $1024$）。

- 公式表示为：

  $$E = Patch \cdot W$$

- 这里的 $E$ 就是所谓的 **Patch Embedding**（图像块嵌入）。

------

### 为什么不直接用像素，非要搞个“投影”？

1. **统一规格**：Transformer 内部的每一层都要求输入向量的维度是固定的（比如都是 768 维）。线性投影能确保不管你输入的 Patch 是大是小，最后出来的向量长度都一致。
2. **提取语义**：原始像素只是颜色亮度。线性投影通过训练，能学会把像素组合成更具代表性的“特征”。比如，它可能学会将“带有斜边的像素组合”转换成特定的数值信号。
3. **计算效率**：通过投影降低或调整维度，可以更好地控制模型整体的计算复杂度。

------

### 进阶小技巧：卷积实现

在代码实现（如 PyTorch）中，工程师们其实很少真的先“打平”再“投影”，因为那样效率很低。

**最优雅的做法是使用一个大的卷积核：**

- 设置卷积核大小（Kernel Size） = $16 \times 16$。
- 设置步长（Stride） = $16$。
- 这样卷积核每跳一次，正好覆盖一个 Patch，直接一步到位提取出特征向量，既快又准。

# vit有什么典型论文必看吗？

想要深入了解 ViT，这几篇论文构成了从“开创”到“优化”再到“全能”的完整路线图。我建议按以下顺序阅读：

### 1. 开山之作：ViT 原型

- **论文：** *An Image is Worth 16x16 Words: Transformers for Image Recognition at Scale* (ICLR 2021)
- **核心内容：** 第一次证明了纯 Transformer 可以在不依赖卷积（CNN）的情况下，在视觉任务上取得 SOTA 效果。
- **必读理由：** 它是所有后续研究的基石，定义了如何将图像切块（Patching）并喂给 Transformer。

### 2. 局部性改进：Swin Transformer

- **论文：** *Swin Transformer: Hierarchical Vision Transformer using Shifted Windows* (ICCV 2021 Best Paper)
- **核心内容：** 引入了“滑动窗口”和“层级化结构”。解决了原生 ViT 对高分辨率图像计算量过大的问题（从平方复杂度降到线性）。
- **必读理由：** 它是目前工业界和竞赛中最常用的骨干网络（Backbone），让 Transformer 在检测和分割任务上超越了 CNN。

------

### 3. 数据效率优化：DeiT

- **论文：** *Training data-efficient image transformers & distillation through attention* (ICML 2021)
- **核心内容：** Facebook 团队提出。ViT 原本需要海量数据（如 JFT-300M），DeiT 通过“蒸馏”技术，让模型在 ImageNet 这种常规数据集上也能练出好结果。
- **必读理由：** 解决了“没钱没数据就玩转不了 Transformer”的痛点。

### 4. 走向全能：MAE (Masked Autoencoders)

- **论文：** *Masked Autoencoders Are Scalable Vision Learners* (CVPR 2022)
- **核心内容：** 何恺明大神的力作。类似 NLP 里的 BERT，它把图片随机盖住 75%，让模型去预测剩下的部分。
- **必读理由：** 视觉自监督学习的巅峰，证明了 Transformer 拥有极强的表征学习能力。

------

### 5. 跨界结合：CLIP

- **论文：** *Learning Transferable Visual Models From Natural Language Supervision* (ICML 2021)
- **核心内容：** OpenAI 提出。将图像和文本映射到同一个特征空间。
- **必读理由：** 它是现在所有 AI 绘图（Stable Diffusion, DALL-E）和多模态模型（GPT-4V）的灵魂。

------
