相关的重要工作包括：

# 2 Open Vocabulary Detection 的核心思想

传统目标检测模型：

```text
训练类别 = 推理类别
```

例如：

```text
训练：cat, dog, car
推理：只能检测 cat, dog, car
```

而 **Open Vocabulary Detection (OVD)** 可以做到：

```text
训练：cat, dog
推理：zebra
```

关键原因是使用了 **视觉-语言共同的 embedding 空间**。

基本流程：

```
image → image encoder → image embedding
text  → text encoder  → text embedding
```

然后通过：

```
cosine similarity
```

判断图像区域与文本的匹配程度。

------

# 3 Cosine Similarity 的含义

余弦相似度衡量的是：

```
两个向量的方向是否一致
```

公式：

[
s = \frac{v \cdot t}{|v||t|}
]

特点：

- 只看 **方向**
- 不看 **向量长度**
- 值域为 **[-1,1]**

解释：

| cos值 | 含义         |
| ----- | ------------ |
| 1     | 完全相同方向 |
| 0     | 正交（无关） |
| -1    | 完全相反     |

在视觉语言模型中：

```
cos(image_embedding, text_embedding)
```

越大，说明语义越相似。

------

# 4 向量方向 vs 向量距离

两个常见概念：

### 1 Cosine Similarity

关注：

```
向量夹角
```

### 2 Euclidean Distance

关注：

```
空间距离
```

在一般情况下：

```
方向一致 ≠ 距离相同
```

例如：

```
v = (1,1)
t1 = (2,2)
t2 = (100,100)
```

cosine：

```
cos(v,t1) = cos(v,t2) = 1
```

但距离不同。

------

# 5 CLIP 中为什么两者等价

在 **CLIP** 中，embedding 会进行：

```
L2 normalization
```

即：

```
v ← v / ||v||
```

结果：

```
所有向量长度 = 1
```

因此所有 embedding 都落在：

```
高维单位球面
```

在这种情况下：

```
cosine similarity
```

与

```
euclidean distance
```

是等价信息。

------

# 6 高维单位球面

embedding 通常是：

```
512维 / 768维
```

单位球面的定义：

```
x₁² + x₂² + ... + xₙ² = 1
```

性质：

- 向量长度固定
- 相似度只由 **方向决定**

高维空间有一个重要性质：

```
随机向量几乎总是接近正交
```

因此在 CLIP 中：

| cosine | 含义       |
| ------ | ---------- |
| 0.05   | 几乎无关   |
| 0.2    | 有一定相关 |
| 0.3    | 很相关     |
| 0.4+   | 非常相关   |

------

# 7 Embedding 对齐（Alignment）

CLIP 的目标是：

```
image embedding
text embedding
```

落入同一个语义空间。

训练前：

```
image space ≠ text space
```

训练后：

```
horse image
≈
"horse"
```

这称为：

```
cross-modal alignment
```

------

# 8 语义地图（Semantic Map）

训练完成后，embedding 空间会形成：

```
语义聚类
```

例如：

```
animal cluster
  ├ horse
  ├ zebra
  └ dog

vehicle cluster
  ├ car
  └ truck
```

这些 cluster 在空间中形成一种：

```
语义地图
```

甚至会出现：

```
层级结构
```

例如：

```
entity
 ├ animal
 │  ├ horse
 │  └ dog
 └ vehicle
```

这种结构并没有被显式标注，而是 **模型从数据统计中自动学到的**。

------

# 9 各向异性（Anisotropy）

真实 embedding 空间并不是均匀球面。

而是：

```
某些方向非常拥挤
某些方向几乎没有向量
```

这称为：

```
embedding anisotropy
```

产生原因：

1. 语言数据统计偏差
2. 对比学习训练目标
3. 高维空间的主方向现象

影响：

- 检索性能下降
- 相似度区分度降低
- OVD 分类困难

------

# 10 Grounding 模型如何解决

像：

- GLIP
- Grounding DINO

会使用：

```
cross-attention
```

实现：

```
text token ↔ image region
```

逐词匹配。

这样可以避免：

```
只依赖 global embedding
```

带来的限制。

------

# 11 今日核心理解

今天最重要的几个概念：

### 1 CLIP 学到的不是分类器

而是：

```
语义空间
```

### 2 embedding 是语义坐标

```
语义相似 → 向量方向接近
```

### 3 OVD 的本质

```
在语义空间中匹配目标
```

### 4 Grounding 的本质

```
在语义空间中找到与文本对应的图像区域
```

------

# 今日一句总结

```
CLIP 并不是在学习“分类”，
而是在学习一个高维的语义地图，
而 Open-Vocabulary Vision 的任务，
本质上是在这个语义地图中寻找目标。
```
